### F2.1 本地向量检索

**边界**：
- 查询文本向量化 → 向量数据库相似度搜索
- 返回 Top-K 结果，包含：文本块、来源文件名、类型、时间、相似度分数
- 支持过滤：按文件类型、时间范围
- 结果去重（同一文件的多个块合并展示）

**不做什么**：不做在线检索

**交付要求**：
- 搜索结果按相似度排序
- 过滤条件生效
- 搜索延迟 < 2 秒（500 文档规模）
- 单元测试覆盖搜索和过滤

**依赖**：F1.7

---

### F2.2 在线资料检索

**边界**：
- 支持至少一个在线检索 Provider（Tavily 或 Bing Search API）
- 仅发送查询关键词，不携带本地文件内容
- 返回：摘要 + 来源 URL
- 用户可配置优先检索域名
- 用户可全局关闭在线检索

**不做什么**：不做多 Provider 切换（预留接口即可）

**交付要求**：
- 在线检索 API 可正常调用并返回结果
- 结果包含摘要和 URL
- 配置开关可控制是否触发
- 单元测试覆盖 API 调用（mock）

**依赖**：F1.1

---

### F2.3 检索结果融合排序

**边界**：
- 统一本地和在线结果的输出格式
- 按相关度分数排序，本地结果优先
- 标注来源类型（本地文件 / 在线资料）
- 支持配置最大返回条数

**不做什么**：不做复杂的重排序算法

**交付要求**：
- 融合结果按规则排序正确
- 来源类型标注清晰
- 单元测试覆盖混合结果场景

**依赖**：F2.1, F2.2

---

### F2.4 知识库文件管理

**边界**：
- 列出已导入文件：文件名、类型标签、时间、索引状态
- 支持按类型、时间范围筛选和排序
- 单文件重新索引
- 删除文件及其关联的索引数据
- 手动修正文件分类标签

**不做什么**：不做批量操作 UI（阶段四实现）

**交付要求**：
- 文件列表可按条件筛选
- 删除操作同时清理向量数据库中的关联数据
- 标签修正后持久化
- 单元测试覆盖 CRUD 操作

**依赖**：F1.7, F1.8
