### F1.1 项目初始化与配置管理

**边界**：
- 创建项目目录结构（后端模块划分）
- 初始化 Conda 虚拟环境（`environment.yml`）
- 初始化 Python 项目依赖声明（`pyproject.toml`）
- 配置文件（config.yaml）定义与读写
- 日志框架搭建

**不做什么**：不涉及具体业务逻辑

**交付要求**：
- 可运行的空项目骨架
- Conda 环境 `doc-assistant` 可正常创建和激活
- config.yaml 包含所有模块的配置项定义
- 日志可正常输出到控制台和文件

**依赖**：无

---

### F1.2 LLM Provider 抽象层

**边界**：
- 定义统一的 LLM 调用接口（chat / classify / embed）
- 实现 Ollama Provider（默认）
- 实现 Claude Provider（可选）
- 支持 Base URL 配置切换
- Provider 工厂模式，根据配置自动选择

**不做什么**：不实现具体业务调用（分类、生成等）

**交付要求**：
- 通过配置切换 Provider，无需改代码
- chat / classify / embed 三个接口可正常调用
- 单元测试覆盖两个 Provider

**依赖**：F1.1

---

### F1.3 压缩包解压与文件格式识别

**边界**：
- 支持 .zip / .rar / .7z 自动解压
- 递归处理嵌套压缩包
- 根据文件扩展名识别格式，过滤不支持的格式
- 返回统一格式的待处理文件列表

**不做什么**：不做文件内容解析

**交付要求**：
- 嵌套压缩包可正确解压到临时目录
- 返回文件列表包含：路径、格式类型、原始压缩包路径
- 单元测试覆盖各压缩格式和嵌套场景

**依赖**：F1.1

---

### F1.4 多格式文档文本提取

**边界**：
- 实现 6 种格式的文本提取器（docx / pdf / xlsx / pptx / png+jpg OCR / txt）
- 每个提取器输出统一格式：纯文本 + 结构信息（标题层级）
- 提取失败的文件记录错误原因，不中断整体流程

**不做什么**：不做分类、分块、向量化

**交付要求**：
- 6 种格式均可正确提取文本
- PDF 扫描件走 OCR fallback
- 输出统一数据结构（text: str, structure: list[dict], metadata: dict）
- 单元测试覆盖各格式，包括异常文件处理

**依赖**：F1.3

---

### F1.5 元数据提取与文档分类

**边界**：
- 提取元数据：文件名、路径、导入时间、文档内日期（正则匹配）
- 文件类型自动分类：调用 LLM 对前 500 字零样本分类
- 分类标签：通知、公告、请示、报告、方案、规划、会议纪要、合同、工作总结、领导讲话稿、调研报告、汇报 PPT、其他
- 支持手动修正分类标签（接口预留）

**不做什么**：不做前端 UI（标签修正在阶段四实现）

**交付要求**：
- 日期正则支持：YYYY年MM月DD日、YYYY-MM-DD、YYYYMMDD、文件名日期
- LLM 分类接口可调用，分类结果写入元数据
- 元数据统一数据结构定义
- 单元测试覆盖日期提取和分类调用

**依赖**：F1.2, F1.4

---

### F1.6 文本分块

**边界**：
- 按段落切分文本，每块 400-600 字
- 相邻块保留 50 字重叠
- 保留每块的元数据关联（来源文件、块序号）

**不做什么**：不做向量化

**交付要求**：
- 分块结果：list[Chunk]，每个 Chunk 包含 text, source_file, chunk_index, metadata
- 边界情况处理：超短文本、超长段落
- 单元测试覆盖正常和边界情况

**依赖**：F1.5

---

### F1.7 向量化与入库

**边界**：
- 调用 Embedding 模型将文本块向量化
- 向量 + 文本块 + 元数据存入本地向量数据库
- 增量更新：记录文件 MD5 + 修改时间，跳过未变更文件
- 文件变更时删除旧数据重新入库

**不做什么**：不做检索功能

**交付要求**：
- 首次导入可完成全流程（解压→提取→分类→分块→向量化→入库）
- 增量导入跳过未变更文件
- 数据库中可查询到完整的文本块和元数据
- 集成测试覆盖全流程

**依赖**：F1.2, F1.6

---

### F1.8 异步导入任务管理

**边界**：
- 导入任务作为异步任务运行
- 每个文件独立处理，失败不中断整体
- 任务状态管理：Pending / Running / Completed / Cancelled / Failed
- 支持查询进度和取消任务
- 进度信息：当前文件、总数、成功/失败/跳过计数

**不做什么**：不做 WebSocket 推送（阶段四实现）

**交付要求**：
- 可启动异步导入任务并查询进度
- 可取消正在运行的任务
- 单文件失败不影响其他文件
- 单元测试覆盖状态流转

**依赖**：F1.7
