### F3.1 意图解析

**边界**：
- 解析用户自然语言描述，提取：文件类型、主题、关键词
- 支持从 12 种公文类型中识别
- 关键词用于触发本地和在线检索

**不做什么**：不做多轮对话

**交付要求**：
- 可从描述中提取文件类型、主题、关键词
- 未识别到文件类型时返回默认值
- 单元测试覆盖典型描述场景

**依赖**：F1.2

---

### F3.2 Prompt 构建

**边界**：
- 动态组装 Prompt 的 5 个部分：角色设定、文风参考、政策依据、格式要求、写作任务
- 角色设定为固定模板
- 文风参考注入 Top-5 检索结果
- 政策依据注入在线检索结果（如有）
- 格式要求从模板库读取
- 支持 Prompt 长度控制，超出时截断优先级：写作任务 > 格式要求 > 文风参考 > 政策依据

**不做什么**：不做 Prompt 优化/AB 测试

**交付要求**：
- 可根据输入和检索结果动态构建完整 Prompt
- 长度超限时按优先级截断
- 单元测试覆盖各组合场景

**依赖**：F3.1, F2.3

---

### F3.3 公文模板管理

**边界**：
- 内置 12 种公文类型模板（通知、公告、请示、报告、方案、规划、会议纪要、合同/协议、工作总结、领导讲话稿、调研报告、汇报 PPT）
- 每个模板包含：结构大纲 + 各段写作要点 + 格式规范
- 用户自定义模板的 CRUD
- 模板存储为 JSON/YAML 格式

**不做什么**：不做前端模板编辑器（阶段四实现）

**交付要求**：
- 12 个内置模板定义完整
- CRUD 接口可正常操作
- 自定义模板与内置模板统一使用
- 单元测试覆盖模板读写

**依赖**：F1.1

---

### F3.4 LLM 公文生成

**边界**：
- 调用 LLM 根据构建的 Prompt 生成公文初稿
- 支持流式输出（token 逐个返回）
- 模式 A：描述 → 自动检索 → 生成
- 模式 B：用户选择素材 + 补充要求 → 生成
- 生成结果附带素材来源清单

**不做什么**：不做 Word 格式化（F3.5 负责）

**交付要求**：
- 模式 A 和模式 B 均可生成完整公文文本
- 流式输出可正常工作
- 素材来源清单完整（文件名 + 时间 + 在线来源 URL）
- 集成测试覆盖两种模式

**依赖**：F3.2, F3.3

---

### F3.5 Word 格式化输出

**边界**：
- 将 LLM 生成的文本格式化为符合 GB/T 9704-2012 的 .docx 文件
- 字体规范：文件标题（方正小标宋简体/二号/居中）、一级标题（方正黑体_GBK/三号/首行空两字）、二级标题（方正楷体_GBK/三号/首行空两字）、三级标题（方正仿宋_GBK/三号/首行空两字）、正文（方正仿宋_GBK/三号/首行空两字）、英文数字（Times New Roman）
- 行距 28 磅，页边距符合国标
- 可选 .md 格式输出
- 文件命名：{文件类型}_{主题关键词}_{生成日期}.docx

**不做什么**：不做 PPT 输出（F3.6 负责）

**交付要求**：
- 输出的 .docx 文件排版符合字体规范表
- 英文/数字在 run 级别使用 Times New Roman
- 文件命名符合规范
- 单元测试覆盖格式化输出

**依赖**：F3.4

---

### F3.6 Word 转 PPT

**边界**：
- 解析 Word 文档结构（提取标题层级、段落要点）
- 调用 LLM 将各章节总结为 PPT 页面内容（标题 + 要点列表）
- 生成 .pptx 文件：封面、目录、各章节页、结语页
- PPT 使用简洁公务风格模板

**不做什么**：不做 PPT 样式自定义

**交付要求**：
- 输入 .docx 可正确生成结构化 .pptx
- PPT 包含封面、目录、章节页、结语页
- 各页内容对应文档结构
- 单元测试覆盖结构解析和生成

**依赖**：F1.2, F3.4
